consumer-async

요청흐름 다이어그램

Online FlowChart & Diagrams Editor - Mermaid Live Editor

[컨슈머 요청 흐름]

1. Consumer.commitAsync(callback)

  • 역할:
    • 사용자 애플리케이션에서 커밋 요청을 시작
    • 오프셋 커밋 작업이 비동기로 수행되며, 커밋 결과를 처리할 콜백(callback)을 제공
  • 발생 위치: 사용자 코드.
  • 결과: 내부적으로 ConsumerCoordinator.commitOffsetsAsync가 호출됨

2. ConsumerCoordinator.commitOffsetsAsync

  • 역할:
    • 비동기 방식으로 오프셋 커밋을 요청한다
    • 내부적으로 오프셋 데이터를 준비하고, 브로커로 전송할 준비시작
  • 발생 위치: ConsumerCoordinator 클래스.
  • 결과: doCommitOffsetsAsync 호출.

3. ConsumerCoordinator.doCommitOffsetsAsync

  • 역할:
    • 커밋할 오프셋 데이터를 검증하고, 네트워크 요청 객체를 생성
    • 브로커로 요청 전송을 준비함
  • 발생 위치: ConsumerCoordinator 클래스.
  • 결과: sendOffsetCommitRequest 호출.

4. ConsumerCoordinator.sendOffsetCommitRequest

  • 역할:
    • OffsetCommitRequest 객체를 생성
    • 요청과 함께 응답을 처리할 핸들러(offsetCommitResponseHandler)를 등록한다
  • 발생 위치: ConsumerCoordinator 클래스.
  • 결과: Consumer.client.send 호출.

5. Consumer.client.send

  • 역할:
    • 네트워크 요청(OffsetCommitRequest)을 Kafka 브로커로 전송한다
    • 응답 핸들러가 네트워크 요청과 함께 등록된다
  • 발생 위치: ConsumerNetworkClient 클래스.
  • 결과: 브로커에 요청 전송.

[브로커 처리 흐름]

6. KafkaApis.handleOffsetCommitRequest

  • 역할:
    • 브로커에서 요청을 수신한다
    • 요청의 유효성을 검사하고, 그룹 코디네이터로 전달
  • 발생 위치: 브로커의 KafkaApis 클래스.
  • 결과: GroupCoordinatorAdapter.handleCommitOffsets 호출.

7. GroupCoordinatorAdapter.handleCommitOffsets

  • 역할:
    • 그룹 코디네이터(GroupCoordinator)의 오프셋 커밋 로직을 호출한다
    • 요청된 오프셋 데이터를 그룹 관리 로직으로 전달한다
  • 발생 위치: 브로커의 GroupCoordinatorAdapter 클래스.
  • 결과: GroupCoordinator.handleCommitOffsets 호출.

8. GroupCoordinator.handleCommitOffsets

  • 역할:
    • 그룹 관련 데이터를 검증하고, 오프셋 커밋 요청을 처리한다
    • _consumer_offsets 토픽으로 오프셋 데이터를 저장하기 위해 준비한다
  • 발생 위치: 브로커의 GroupCoordinator 클래스.
  • 결과: GroupCoordinator.doCommitOffsets 호출.

9. GroupCoordinator.doCommitOffsets

  • 역할:
    • _consumer_offsets 토픽에 기록할 데이터를 준비한다
    • 데이터를 저장하기 위해 ReplicaManager.appendRecords를 호출한다
  • 발생 위치: 브로커의 GroupCoordinator 클래스.
  • 결과: ReplicaManager.appendRecords 호출.

10. ReplicaManager.appendRecords

  • 역할:
    • _consumer_offsets 토픽의 리더 파티션에 데이터를 저장한다
    • ISR(In-Sync Replica)에 복제를 수행함
  • 발생 위치: 브로커의 ReplicaManager 클래스.
  • 결과: 저장 완료 후 응답 콜백 호출.

11. responseCallback 호출 (오프셋 저장 결과)

  • 역할:
    • _consumer_offsets에 오프셋이 저장된 결과를 처리한다
    • 저장 성공 또는 실패 여부를 상위 호출자로 전달한다
  • 발생 위치: GroupCoordinator.doCommitOffsets.
  • 결과: GroupCoordinator 계층의 콜백 호출.

12~14. GroupCoordinator 계층의 responseCallback 처리

  • 역할:
    • 응답 콜백을 통해 저장 성공 여부를 상위 호출자에게 전달한다
    • 최종적으로 KafkaApis에 결과를 반환한다
  • 발생 위치: GroupCoordinator 및 GroupCoordinatorAdapter.
  • 결과: KafkaApis.handleOffsetCommitRequest로 결과 전달.

15. KafkaApis.handleOffsetCommitRequest에서 OffsetCommitResponse 생성

  • 역할:
    • 저장 결과를 기반으로 응답 객체(OffsetCommitResponse)를 생성한다
    • 클라이언트로 반환할 준비를 완료
  • 발생 위치: 브로커의 KafkaApis.
  • 결과: KafkaApis.sendResponseMaybeThrottle 호출.

16. KafkaApis.sendResponseMaybeThrottle

  • 역할:
    • 클라이언트 응답 전 스로틀링을 적용할지 확인한다
    • 필요 시 응답을 지연시킨다
  • 발생 위치: 브로커의 KafkaApis.
  • 결과: KafkaApis.sendResponse 호출.

17. RequestChannel.sendResponse

  • 역할:
    • 브로커가 응답 데이터를 클라이언트로 전송한다
  • 발생 위치: 브로커의 RequestChannel.
  • 결과: 클라이언트로 응답 전송 완료.

[컨슈머 응답 처리]

18. Consumer.client.poll()로 응답 수신

  • 역할:
    • 컨슈머가 브로커의 응답을 수신한다
    • 등록된 핸들러를 실행합니다.
  • 발생 위치: ConsumerNetworkClient 클래스.

19. offsetCommitResponseHandler 호출

  • 역할:
    • 응답 데이터를 처리하고, 성공/실패를 판별한다
  • 발생 위치: ConsumerCoordinator.
  • 결과: 사용자 콜백 호출.

20. 사용자 콜백(callback) 호출

  • 역할:
    • 사용자가 commitAsync 호출 시 등록한 콜백을 호출한다
    • 커밋 성공/실패 결과를 애플리케이션에 전달한다
  • 발생 위치: 사용자 코드.
